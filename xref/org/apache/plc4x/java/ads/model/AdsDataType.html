285      * Pointer
286      * Variable or function block addresses are saved in pointers while a program is running. Pointer declarations have the following syntax:
287      * <p>
288      * &lt;Identifier&gt;: POINTER TO &lt;Datatype/Functionblock&gt;;
289      * A pointer can point to any data type or function block even to user-defined types. The function of the Address Operator ADR is to assign the address of a variable or function block to the pointer.
290      * A pointer can be dereferenced by adding the content operator "^" after the pointer identifier. With the help of the SIZEOF Operator, e.g. a pointer increment can be done.
291      * <p>
292      * <p>
293      * Please note: A pointer is counted up byte-wise ! You can get it counted up like it is usual in the C-Compiler by using the instruction p=p+SIZEOF(p^);.
294      * <p>
295      * <p>
296      * <p>
297      * Attention:
298      * After an Online Change there might be changes concerning the data on certain addresses. Please regard this in case of using pointers on addresses.
299      * <p>
300      * <p>
301      * <p>
302      * Example:
303      * <p>
304      * pt:POINTER TO INT;
305      * var_int1:INT := 5;
306      * var_int2:INT;
307      * <p>
308      * <p>
309      * pt := ADR(var_int1);
310      * var_int2:= pt^; (* var_int2 is now 5 *)
311      * <p>
312      * <p>
313      * Example 2 (Pointer increment):
314      * <p>
315      * ptByCurrDataOffs : POINTER TO BYTE;
316      * udiAddress       : UDINT;
317      * <p>
318      * <p>
319      * <p>
320      * (*--- pointer increment ---*)
321      * udiAddress := ptByCurrDataOffs;
322      * udiAddress := udiAddress + SIZEOF(ptByCurrDataOffs^);
323      * ptByCurrDataOffs := udiAddress;
324      * (* -- end of pointer increment ---*)
325      */

328      * Enumeration (ENUM)
329      * Enumeration is a user-defined data type that is made up of a number of string constants. These constants are referred to as enumeration values. Enumeration values are recognized in all areas of the project even if they were locally declared within aPOU. It is best to create your enumerations as objects in the Object Organizer under the register card Data types. They begin with the keyword TYPE and end with END_TYPE.
330      * <p>
331      * Syntax:
332      * <p>
333      * TYPE &lt;Identifier&gt;:(&lt;Enum_0&gt; ,&lt;Enum_1&gt;, ...,&lt;Enum_n&gt;);END_TYPE
334      * <p>
335      * The &lt;Identifier&gt; can take on one of the enumeration values and will be initialized with the first one. These values are compatible with whole numbers which means that you can perform operations with them just as you would with INT. You can assign a number x to the &lt;Identifier&gt;. If the enumeration values are not initialized, counting will begin with 0. When initializing, make certain the initial values are increasing. The validity of the number will be reviewed at the time it is run.
336      * <p>
337      * Example:
338      * <p>
339      * TRAFFIC_SIGNAL: (Red, Yellow, Green:=10); (*The initial value for each of the colors is red 0, yellow 1, green 10 *)
340      * TRAFFIC_SIGNAL:=0; (* The value of the traffic signal is red*)
341      * FOR i:= Red TO Green DO
342      * i := i + 1;
343      * END_FOR;
344      * <p>
345      * You may not use the same enumeration value more than once.
346      * <p>
347      * Example:
348      * <p>
349      * TRAFFIC_SIGNAL: (red, yellow, green);
350      * COLOR: (blue, white, red);
351      * <p>
352      * Error: red may not be used for both TRAFFIC_SIGNAL and COLOR.
353      */

376      * Point2:ARRAY [1..2] OF INT;

518         this.lowerBound = lowerBound;
519         this.lowerBoundDouble = lowerBound.doubleValue();
520         this.upperBound = upperBound;

522         this.typeName = name();
523         this.memoryUse = memoryUse;
524         this.targetByteSize = this.memoryUse / 8;
525     }

562     private static BigInteger produceUnsignedMaxValue(int numberOfBytes) {
563         return new BigInteger(
564             ArrayUtils.insert(
565                 0,
566                 IntStream.range(0, numberOfBytes)
567                     .map(ignore -> 0xff)
568                     .collect(
569                         ByteArrayOutputStream::new,
570                         (baos, i) -> baos.write((byte) i),
571                         (baos1, baos2) -> baos1.write(baos2.toByteArray(), 0, baos2.size())
572                     )
573                     .toByteArray(),
574                 (byte) 0x0)
575         );
